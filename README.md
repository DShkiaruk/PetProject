# Kartka

Приватний **сімейний медичний щоденник** для українських сімей. Одне місце для
всієї медичної інформації родини — без хаосу з паперами.

> Прибрати ризик «загубив виписку / забув ліки / не пам'ятаю, що призначали» і
> дати змогу за секунди поділитися повним анамнезом з лікарем.

Це **design-engineering репозиторій**: тут проходить весь шлях від дослідження
до готового до коду дизайну. Цей README — **живий індекс**: тримати в
актуальному стані разом зі структурою.

## Продукт коротко

**Для кого:** дорослі, що ведуть здоров'я родини — діти та/або літні батьки.
Цінують менше паперового хаосу, приватність і готовність показати все лікарю за
секунди.

**Фічі MVP:**

1. Профілі членів сім'ї + базова картка (алергії, хроніка, група крові).
2. Документи (виписки, результати) як фото/файли з полями: дата, лікар, тип.
3. Трекінг ліків по кожному члену сім'ї: дозування, період, історія.
4. Трекінг аналізів з історією за параметром у часі (графіки).
5. Нагадування про прийоми до лікарів, зокрема повторювані.
6. Експорт анамнезу в PDF для швидкого шерінгу з лікарем.
7. Екстрений екран — усе критичне на одному екрані. + опційно: deep-link «Знайти лікарню/травмпункт поряд» (зовнішні мапи, без даних пацієнта; поза основним MVP).

**Принцип №1:** дані вводяться вручну → введення має бути максимально швидким
(мінімум полів, розумні дефолти, шаблони, повтор останнього запису).

**Межі:** не медичний продукт (жодних діагнозів/порад/норм); приватність
(шифрування at-rest, ізоляція через RLS, GDPR).

## Стек (фаза коду)

- **Next.js** (App Router) на **Vercel**
- **Supabase** — Postgres (RLS), Auth, Storage
- **PWA** (mobile-first, встановлення на домашній екран)
- PDF — серверна генерація зі збережених даних
- Мови: українська + англійська

Платформа: мобільний веб / PWA, далі адаптив до десктопу.
