# Болі користувачів — Kartka

Реальні болі з чужих відгуків як **проксі замість живих інтерв'ю**. Мета —
витягти патерни: що дратує, чому кидають і що тримає продукти з тим самим JTBD,
що в Kartka (веду медичну інформацію родини).

---

## 1. Метод і джерела

**Як збирав:** web-пошук + читання зведень із App Store / Google Play,
Trustpilot, G2, профільних форумів (MacRumors, Mayo Clinic Connect), новинних
матеріалів і UX-кейсів. Пріоритет — негатив (1–2★) і середні (3★); додатково
зібрано похвалу (5★) для виявлення must-have.

**Продукти:** Medisafe, Helsi, лабораторії (Synevo, ДІЛА, Esculab), Apple
Health, MyChart.

**Обсяг сигналу (чесно):**

| Продукт | Сигнал | Примітка |
|---|---|---|
| Medisafe | 🟢 сильний | Trustpilot + app-stores + огляди; 4.5★ |
| MyChart | 🟢 сильний | великі датасети відгуків, Trustpilot/G2 |
| Apple Health | 🟡 середній | форуми + App Store |
| Helsi | 🟡 середній | переважно новини/статус-сервіси, мало структурованих app-store текстів |
| Synevo / ДІЛА | 🟡 середній | один якісний UX-кейс (Promodo) + FAQ; прямих відгуків мало |
| Esculab | 🔴 мало сигналу | майже немає публічних відгуків — **не додумую** |
| PicnicHealth | 🔴 мало сигналу | сервіс, не масовий застосунок — відгуки не збирав |

**Обмеження:**
- Частина «цитат» — це **перекази відгуків** пошуковим шаром, не гарантовано
  дослівні. Де нижче стоїть лапка — це **ілюстративна фраза**, не точна цитата.
- App Store/Google Play фільтрують через US-видачу — UA-специфіка недопредставлена.
- `[факт]` — прямо з відгуків/джерел; `[інтерпретація]` — мій висновок.

## 3. ТОП-патерни болю (наскрізні, за частотою)

> Відсортовано за тим, у скількох продуктах патерн повторюється.

**1. Фрагментація: дані розкидані, немає «одного місця»** — `[факт]`
MyChart «не universal record» → неповні/дубльовані записи й недовіра; Helsi/лаби
сило по провайдеру; на Mayo-форумі пацієнти самі пишуть «one-page summary» для
лікаря. (MyChart, Helsi, лаби, Mayo — 4 джерела)
→ **Kartka:** наш стрижень — приватна крос-провайдерна агрегація; це
підтверджений біль, не гіпотеза.

**2. Навігація й перевантаженість: не знайти ключове** — `[факт]`
Apple «cluttered», «забагато кроків»; Synevo ~70% виходів і не знайти «динаміку»;
MyChart «забагато категорій → пошук». (Apple, Synevo, MyChart — 3)
→ **Kartka:** головний екран = 3–5 дій; «динаміка показника» має бути за 1 тап,
не захована.

**3. Нагадування зроблені погано (надсилають занадто / не контролюються)** — `[факт]`
Medisafe «кожні 5 хв», дублі між пристроями, жорсткі «4 коробки».
→ **Kartka:** нагадування мають бути керованими (інтервал, тиша, підтвердження
синхронне між пристроями) — інакше це anti-feature.

**4. Тертя ручного введення й неможливість редагувати** — `[факт]` (Apple) + `[інтерпретація]`
Apple: правка часу збиває інше поле; «лише видалити весь запис». Більшість
конкурентів узагалі **уникають** ручного вводу (автозбір) — а в нас він основний.
→ **Kartka:** введення й **редагування** мусять бути швидкими й прощати помилки;
це наш екзистенційний ризик №1.

**5. Зрада ціною: «був безкоштовний → тепер платить»** — `[факт]`
Найсильніший churn-тригер у Medisafe; «charges incredibly vulnerable users».
→ **Kartka:** не замикати базове ведення здоров'я за раптовий paywall; якщо буде
монетизація — чесна межа з 1-го дня, базовий архів безкоштовний.

**6. Логін / автентифікація як бар'єр** — `[факт]`
MyChart: 2FA-помилки, не зайти, кілька акаунтів.
→ **Kartka:** вхід простий і надійний; відновлення доступу продумане (ми ж обрали
at-rest саме заради відновлення).

**7. Недовіра до цілісності даних** — `[факт]` (2)
Helsi «фейкові прийоми»; MyChart дублі/неповнота → «mistrust».
→ **Kartka:** користувач сам — джерело правди; показувати походження запису й
давати легко виправити; не «вигадувати» дані.

**8. Надійність, збої, втрата даних** — `[факт]` (2)
Helsi «не відкривається весь день»; Medisafe апдейти стирали ліки.
→ **Kartka:** офлайн-доступ до перегляду критичного (екстрений екран) і захист
від втрати при синхронізації.

**9. Сімейний доступ: або плутає, або тримає** — `[факт, змішано]`
MyChart кілька акаунтів плутають (anti-pattern); натомість Medisafe Dependents і
ДІЛА «вся родина» — хвалять.
→ **Kartka:** один акаунт + профілі (наш вибір) — підтверджено правильним; уникати
моделі «окремий акаунт на кожного».

**10. Слабкий мобільний UX** — `[факт]` (2)
Synevo низька мобільна продуктивність; Apple «забагато кроків».
→ **Kartka:** mobile-first — не гасло, а вимога; кожен зайвий крок = відтік.

## 4. Feature gaps, які люди прямо просять

- **Редагувати запис, а не видаляти весь** (Apple Health). `[факт]`
- **Налаштовувані вікна/тиша нагадувань**, не «кожні 5 хв» (Medisafe). `[факт]`
- **Єдиний record на всіх провайдерів** (MyChart «not universal»). `[факт]`
- **Об'єднати профілі/родину без втрати даних** (MyChart видаляє при merge). `[факт]`
- **Легко знайти «динаміку показника»** в часі (Synevo). `[факт]`
- **«One-page summary» для лікаря** — пацієнти роблять його вручну (Mayo). `[факт]`
  → це **буквально наш PDF-анамнез**.

## 5. Болі української аудиторії / сценарію родини

- **Обіцянка ЕСОЗ vs реальність.** Офіційно: внесене в ЕСОЗ означає «пацієнту не
  треба зберігати паперові виписки». Але на практиці — збої Helsi, «фейкові
  прийоми», фрагментація. `[факт]` → `[інтерпретація]`: довіра до єдиного
  держджерела хитка → є місце для **приватного особистого архіву** під контролем
  користувача.
- **Сімейність уже очікувана.** Helsi «Пов'язані особи», ДІЛА замовлення «на всю
  родину» — українці звикли вести рідних. `[факт]` → профілі родини це must, не
  diff.
- **Аналізи в UA подані слабко.** UX-кейс Synevo (70% виходів, не знайти
  динаміку) — пряме свідчення. `[факт]` → гарна історія параметра в часі —
  реальний диференціатор на нашому ринку.
- **Папір не зникає.** Юридично історія хвороби зберігається в установах до 25
  років, але **копії пацієнт отримує лише за запитом** — тертя доступу. `[факт]`
  → особистий архів знижує залежність від «піти взяти виписку».
- **Літні батьки.** Емоційні історії доглядальників (Medisafe «saved my mom's
  life») показують: нагадування + видимість «хто що приймає» резонують саме в
  сценарії догляду. `[факт, інший ринок]` → `[інтерпретація]`: переноситься на
  UA-сім'ї, що ведуть літніх.

## 6. Зв'язок із відкритими питаннями (`competitors.md`)

| # | Питання | Вердикт із відгуків |
|---|---|---|
| 1 | Введення швидше за «фото в Telegram»? | ✅ **Підтверджено критичним** — тертя вводу/редагування реально женить (Apple). Швидкість вводу = виживання. |
| 2 | Інтегрувати ЕСОЗ/Helsi чи бути автономним? | ↪️ Схиляє до **автономного** архіву: ЕСОЗ/Helsi ненадійні (збої, фейкові записи); інтеграція — опція, не фундамент. |
| 3 | Мінімум метаполів? | ✅ **Підтверджено** — MyChart «забагато категорій → пошук». Менше полів + сильний пошук. |
| 4 | Історія параметра при гібриді? | ✅ Тренди — must (Apple хвалять), але **discoverability — ризик** (Synevo не знаходили динаміку). |
| 5 | Модель сімейного доступу? | ✅ **Один акаунт + профілі валідовано**; кілька акаунтів (MyChart) — anti-pattern. |
| 6 | Формат PDF-анамнезу для UA-лікарів? | 🟡 Попит доведено (Mayo «one-page summary» вручну), але **конкретний UA-формат — досі відкритий**. |
| 7 | Push-фолбек (email/локальні)? | 🟡 Частково: головне — **керованість** нагадувань (бунт проти Medisafe-спаму), а не канал. |
| 8 | Екстрений екран на lock-screen? | ⬜ Немає прямого сигналу — **лишається відкритим**. |
| 9 | Межа «не медичний» при трендах без норм? | ⬜ Прямого сигналу немає; Apple «highlights/insights» люблять — стежити, щоб не зісковзнути в інтерпретацію. |
| 10 | Що несуть лікарю сьогодні? | ✅ Самописні summary (Mayo), результати «надіслати лікарю» (Helsi), паперова папка — **підтверджує цінність PDF-анамнезу**. |
