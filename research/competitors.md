## Поточний стан (верифікація, червень 2026)

- **CareZone** — **згорнутий**. Walmart купив технологію за ~$200М (2020),
  застосунок закрито близько 2023. Лишаємо як **архівний еталон** ідеї
  «сімейний органайзер навколо ліків» (фото банки → список, профілі рідних).
- **Medisafe** — **активний**. Сайт `medisafe.com` зараз — B2B «Patient
  Engagement Platform for Pharma»; консьюмерський контент живе на
  `medisafeapp.com`. Модель: freemium ($4.99/міс, $39.99/рік) + продаж
  engagement фармі.
- **PicnicHealth** — **активний**. AI-агрегація медзаписів у таймлайн;
  монетизація — анонімізовані дані для фарми/досліджень.
- **Helsi** — **активний, домінує в UA**. 8.3М+ завантажень; «Пов'язані особи»
  (діти/рідні), інтеграція з ЕСОЗ. Є публічні скарги на фіктивні записи в ЕМК
  (mind.ua) — сигнал про ризик цілісності автозібраних даних.
- **DILA / Synevo / Esculab** — активні лабораторні кабінети; дані сило по
  одному провайдеру.

## Внутрішні UX-патерни (зі знятих/побачених екранів)

> `[факт]` — спостережений патерн у продукті; **→** — висновок/рішення для Kartka.

1. **Medisafe — приватність на lock-screen.** `[факт]` Нагадування показується
   **без назви ліків**; swipe → «Take All / Snooze All / dismiss».
   **→** Нагадування й екстрений показ не світять діагноз/ліки на замку.
2. **Medisafe — додавання ліків.** `[факт]` Пошук назви в базі → форма/колір
   пігулки (візуальний ID) → частота + **«Days limit»** для курсів (антибіотик) →
   можна пропустити нагадування для «за потреби».
   **→** Пресети тривалості (курс vs постійно) + іконка швидші за вільний текст.
3. **Medisafe — підтвердження прийому.** `[факт]` Кругла «пігулниця», тап =
   прийняв, **undo**, олівець для правки вже залогованого.
   **→** Запис мусить мати миттєвий undo і правку, а не лише видалення (прямий
   контраст із болем Apple «лише видалити весь запис»).
4. **Medisafe — MedFriend / Dependents.** `[факт]` Запрошення по email-лінку; алерт
   другу через 30 хв після пропуску (без назв ліків); person-icon → додати
   залежного = другий профіль.
   **→** Перемикач профілів родини через аватар; майбутній «опікун» по інвайту.
5. **Apple — графік показника.** `[факт]` Тап на показник → інтерактивна крива в
   часі + Highlights/Trends.
   **→** Тап на аналіз → крива; але **без оцінкових highlights** (ми не медпродукт).
6. **Apple — Medical ID на замку.** `[факт]` Критичне (алергії, ліки, група крові)
   видно **без розблокування** + під час екстреного дзвінка.
   **→** Наш екстрений екран має бути доступним без повного входу (PWA-обмеження
   дослідити — це наше відкрите питання Q8).
7. **Apple — Sharing tab.** `[факт]` Окрема вкладка; вибір **конкретних** метрик для
   шерінгу з людиною/лікарем.
   **→** Шерінг вибірковий за профілем/розділом, не «все або нічого».
8. **MyChart — proxy/родина.** `[факт]` Перемикання особи через дропдаун у шапці; 3
   типи proxy (дитина / підліток / дорослий-дорослий); екран згоди-дисклеймера.
   **→** Підтверджує «перемикач профілю»; **згода — окремий явний крок**.
9. **MyChart — лаб-тренди + «Share Everywhere».** `[факт]` Показник на графіку
   відносно цільового числа; разовий **код доступу** для будь-якого лікаря.
   **→** Наш PDF-анамнез = офлайн-аналог «share everywhere» (документ замість коду).
10. **Synevo (UA) — антипатерн.** `[факт]` До редизайну ~70% виходів зі сторінки
    результатів, користувачі **не знаходили «динаміку»**.
    **→** «Динаміка показника» — первинна дія на екрані аналізу, не похована вглиб.
11. **Пошук закладу поряд.** `[факт]` Helsi — карта медзакладів (ЕСОЗ+запис);
    MyChart «On My Way» — найближчі ER/urgent **з часом очікування**; Apple SOS —
    лінк на мапи; ДІЛА — локатор власних відділень.
    **→** Для нас — **зовнішній handoff у мапи ОС** на екстреному екрані, без
    власного каталогу/тріажу. Деталі: [`ux-patterns.md`](./ux-patterns.md) §I.

## 10 спільних патернів

1. **Таймлайн медичних подій** як головна метафора (PicnicHealth, Apple, MyChart).
2. **Графік показника в часі** для аналізів — клік на значення → крива (PicnicHealth, Apple, DILA).
3. **Профілі / перемикання між членами родини** («Пов'язані особи», Dependents, proxy-доступ) — у всіх серйозних.
4. **Нагадування** (ліки/візити) як головний retention-гачок (Medisafe, Helsi, Apple).
5. **Шерінг з лікарем/родиною кількома тапами** (Apple Health Sharing, PicnicHealth, MyChart).
6. **Приватність/шифрування — головний меседж довіри** у кожного health-records продукту.
7. **Безкоштовно для пацієнта; гроші деінде** (фарма / держава / залізо / клініка).
8. **Автозбір даних, щоб уникнути ручного введення** (Apple-сенсори, PicnicHealth-агрегація, Helsi-ЕСОЗ).
9. **Зведений «one-glance» екран** (Apple Medical ID, дашборди) ≈ наш екстрений екран.
10. **Документи/результати як прикріплені файли** + кілька метаполів (дата / тип / джерело).

## 10 відмінностей

1. **Джерело даних:** автоматичне з системи (Helsi/ЕСОЗ, MyChart/EHR, Apple/сенсори, PicnicHealth/агрегація) vs **ручне** (Kartka, нотатки).
2. **Прив'язка до провайдера:** MyChart/DILA/Helsi прив'язані до клініки/лабораторії/держави; Kartka/Apple/PicnicHealth — провайдер-незалежні.
3. **Роль AI:** PicnicHealth (і дедалі більше інших) спираються на AI для оцифрування — **Kartka свідомо без AI**.
4. **Бізнес на даних:** PicnicHealth/Medisafe монетизують дані/engagement фармі — **Kartka принципово не передає дані**.
5. **Ринок:** Apple/Medisafe/PicnicHealth/MyChart — US/глобал; Helsi/DILA — Україна; Kartka — Україна з міжнародним рівнем UX.
6. **Фокус:** Medisafe — лише ліки; DILA — лише аналізи; Kartka/Apple — широкий сімейний архів.
7. **Платформа:** Apple — нативний iOS/залізо; Kartka — **PWA, крос-платформа**.
8. **Глибина «сімейності»:** CareZone/Medisafe/Helsi мають родину; багато інших — суто індивідуальні.
9. **Модель приватності:** Apple — on-device/E2EE; решта — хмара server-side; Kartka — server-side at-rest **із планом E2EE**.
10. **«Анамнез одним документом»:** сильний у PicnicHealth; у локальних UA фрагментовано/відсутній — **це наш отвір**.

## 10 відкритих питань (для нас)

1. Як зробити ручне введення **швидшим за «сфоткати в Telegram»** — конкретні патерни (повтор останнього, шаблони, OCR без AI?).
2. Інтегруватися/доповнювати ЕСОЗ-Helsi (імпорт) чи лишатися повністю автономним сховищем?
3. Який **мінімальний набір метаполів** документа, щоб пошук працював, але введення не дратувало?
4. Як подати **історію параметра в часі** при гібридній моделі (різні одиниці, ручні значення)?
5. Сімейний доступ: MVP — «один акаунт», але як спроєктувати під майбутні «Пов'язані особи» без переписування?
6. Який формат **PDF-анамнезу лікарі реально приймають в Україні** (мова, структура, обсяг)?
7. Нагадування без нативного push на iOS-PWA — наскільки життєздатний **email/локальний фолбек** як основний канал?
8. Чи потрібен **екстрений екран без розблокування** (lock-screen) — і чи можливо це в PWA?
9. Де межа «не медичний продукт» при показі трендів аналізів **без норм**, щоб не виглядати як інтерпретація?
10. Що користувачі **реально приносять лікарю сьогодні** (папір/фото) — і що з цього must-have у v1?
