## 1. Користувачі та болі

**Стрижневий JTBD** `[факт, бриф+jtbd.md]`: коли хтось у родині має справу з
лікарем/ліками/аналізами — мати всю медінформацію в одному приватному місці й
діставати потрібне за секунди. Аудиторія: дорослий-**Координатор** 2–4 профілів
(діти + літні батьки). → [`jtbd.md`](./jtbd.md) (10 сценаріїв → вимоги до екранів).

**Найгостріші підтверджені болі** (з [`user-pains.md`](./user-pains.md)):
- **Фрагментація — немає «одного місця»** `[факт]` (MyChart/Helsi/лаби/Mayo, 4 джерела). Це наш стрижень як реальний біль.
- **Тертя ручного введення/правки → відтік** `[факт]` (Apple Health: «лише видалити весь запис»). **Ризик №1.**
- **Навігація/перевантаження — «не знайти ключове»** `[факт]` (Apple, Synevo ~70% виходів, MyChart «resort to search»). **Біль №2.**
- **Зрада ціною** `[факт]` (Medisafe paywall — головний churn-тригер).
- **Недовіра до цілісності даних** `[факт]` (Helsi «фейкові прийоми»; MyChart дублі).
- **Попит на «анамнез одним документом»** `[факт]` (Mayo: пацієнти вручну пишуть one-page summary).

→ `[інтерпретація]`: довіра до єдиного держджерела (ЕСОЗ) хитка → є місце для
приватного автономного архіву під контролем користувача.

## 2. Конкуренти / ринок

Повна матриця за осями (аудиторія / основа / механізм / довіра / монетизація) +
3 групи (хард/софт/аспіраційні) — у [`competitors.md`](./competitors.md).

**Спільні патерни ринку** `[факт]`: таймлайн подій · графік показника в часі ·
профілі родини · нагадування як retention-гачок · шерінг кількома тапами ·
приватність як меседж довіри · безкоштовно для пацієнта (гроші деінде) · автозбір
даних, щоб уникнути ручного вводу.

**Ключові відмінності / наш отвір** `[факт+інтерпретація]`:
- Майже всі **уникають ручного вводу** (автозбір/сенсори/агрегація) — Kartka навмисно ручна → швидкість вводу = виживання.
- **«Анамнез одним рухом, крос-провайдер»** сильний лише в PicnicHealth (US, через AI); у локальних UA — фрагментовано/відсутній → **наш отвір**.
- Ми єдині поєднуємо: UA-ринок + без AI + без монетизації даними + PWA + сімейні профілі.

## 3. Бенчмарк → механізми в MVP

Best-in-class із будь-яких категорій, по 3 вимірах (повні таблиці 1–5 у
[`benchmark.md`](./benchmark.md)):

| Вимір | БЕРЕМО в MVP | НЕ спрацює (і чому) |
|---|---|---|
| **Швидке введення** | мінімум обов'язкових полів + дефолти (Spendee); «повтор останнього» (MyFitnessPal); завжди-«+» bottom-sheet (Things) | **повний NLP-рядок** (Todoist) — медназви/дозування ненадійні без AI (заборонено), укр. морфологія, ціна помилки висока |
| **Анамнез одним рухом** | кнопка-експорт завжди (Resume.io); вибір розділів + чистий шаблон (Apple/Revolut); офлайн-PDF у будь-який канал (Revolut) | **жива авторизована стрічка** (Apple Sharing) — важкий онбординг, лише орг-учасниці (немає в UA), не офлайн |
| **Профілі** | аватар-світчер 2 тапи (Netflix/Google); бейдж+колір+підсвітка активного; підтвердження «для кого» | **окремі акаунти + proxy на кожного** (MyChart) — плутає, важка згода, злиття губить дані |

## 4. Інформаційна архітектура (рішення)

**Обрано: профіль-перший каркас + дашборд «важливе» як домівка профілю**
`[рішення]` (обґрунтування — [`patterns-information-architecture.md`](./patterns-information-architecture.md)).
Чому під наш контекст: родина — структурна вісь (знижує «внести не тому»); дашборд
б'є біль №2 і живить екстрений екран + вхід у PDF; профіль згори → менше полів на
вводі (не конфліктує з ризиком №1). Типи (ліки/аналізи/документи) — розділи під
drill-down; **пошук — вторинний** (escape hatch, не основа: холодний старт +
тонкі метадані + без AI). **Таймлайн — другий**; стає першим, **якщо** зʼявиться
автоімпорт/інтеграція (стрічка наповнюється сама).

## 5. Рішення зі скоупу (ключові межі)

Джерело — [`../CLAUDE.md`](../CLAUDE.md) (+ [`ux-patterns.md`](./ux-patterns.md) §I).
- **Не медичний продукт:** жодних діагнозів/порад/норм; аналізи — динаміка **без референсних норм**.
- **Приватність:** хмара + шифрування at-rest, ізоляція через **RLS** (не `where` у коді); план E2EE для найчутливіших полів.
- **Без зовнішнього AI** і **без передачі медданих третім**.
- **Зовнішні заклади** («лікарня поряд») — **лише deep-link у мапи ОС**, без власного каталогу й тріажу; передається лише анонімний геозапит. `[факт, CLAUDE.md/ux-patterns §I]`
- **Доступ:** один акаунт на домогосподарство + профілі (multi-user/опікун — поза MVP).

## 7. Висновки: gaps і гіпотези («якщо / то / бо»)

1. **Якщо** зробити додавання документа фото-першим із ≤3 тапами, **то** знімемо ризик №1, **бо** тертя вводу/правки прямо женить (Apple Health, [`user-pains.md`](./user-pains.md)) і best-in-class роблять мінімум полів ([`benchmark.md`](./benchmark.md) В1).
2. **Якщо** профіль-перший + підсвітка активного на кожному екрані вводу, **то** знизимо «внести не тому», **бо** кілька акаунтів плутають (MyChart, user-pains) і family-switch — must (benchmark В3).
3. **Якщо** PDF-анамнез = самодостатній офлайн-документ із вибором розділів, **то** закриємо головний диференціатор, **бо** попит доведено (Mayo one-page summary) і жива стрічка не працює в UA (benchmark В2, competitors).
4. **Якщо** «динаміка показника» = первинна дія на екрані аналізу, **то** уникнемо болю «не знайти», **бо** Synevo втрачав ~70% на захованій динаміці ([`ux-patterns.md`](./ux-patterns.md) §B/Synevo).
5. **Якщо** нагадування керовані (інтервал/тиша, без назв на замку), **то** уникнемо anti-feature, **бо** агресивний спам — головний churn Medisafe (user-pains).
6. **Гіпотеза-ризик:** **якщо** лишитися автономними від ЕСОЗ, **то** виграємо в надійності/довірі (Helsi збої+фейкові записи), **але бо** ввід ручний — лишається ризик **холодного старту** → перевірити на прототипі.
